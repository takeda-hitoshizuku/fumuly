# fumuly - 収納用バーコード支払い連携機能 実装仕様

## 概要

払込票（公共料金・クレジット会社など）に印刷された**コンビニ収納用バーコード**を読み取り、PayPayやPayBなどの外部支払いアプリへ連携するテスト機能。

ステータス: **β実装（テスト機能）**

---

## 背景と技術的制約

### なぜアプリ内で完結できないか

払込票のバーコードによる支払いは **資金移動業** に該当する。PayBやPayPayがこれを実現できるのは金融庁への登録ライセンスを持つためであり、個人開発アプリでの直接実装は法的・技術的に不可能。

### バーコードの仕組み

払込票に印刷されているのは「コンビニ収納用バーコード」。振込先口座番号ではなく、収納代行業者（電算システム・GMOペイメントゲートウェイ等）との契約企業が発行したバーコードに支払い情報がエンコードされている。

```
バーコード数値（13桁または20桁）
  └ 収納機関番号 + 金額 + 確認番号 が含まれる
```

### 「対応していません」が多い理由

PayBやモバイルレジで弾かれるのは、払込票を発行している企業・団体が収納代行業者の**加盟企業になっていない**ため。バーコードがあっても未加盟なら支払い不可。fumuly でもこの制約は変わらない。

---

## 実装方針

「支払いをアプリ内で完結させる」ではなく、「**ユーザーが外部アプリで支払いやすくする**」設計。

```
fumuly でバーコード数値を抽出
      ↓
「外部アプリで支払う (β)」ボタンを書類詳細に表示
      ↓
PayPay / PayB のディープリンクで対応アプリを起動
      ↓
ユーザーが確認・支払い完了
      ↓
fumuly に戻り「支払い済み」マークを手動セット
```

ユーザーは**再スキャン不要・コンビニ不要**で支払いを試みることができる。

## Claude Vision プロンプト変更

書類解析時にバーコード数値も抽出するよう指示を追加する。

### 追加する抽出フィールド

```typescript
interface DocumentAnalysis {
  // 既存フィールド
  sender: string;
  documentType: string;
  amount: number;
  dueDate: string;
  priority: 'urgent' | 'action' | 'keep' | 'ignore';
  summary: string;
  recommendedActions: string[];

  // 追加フィールド
  barcodeNumber?: string;      // 収納用バーコードの数値列（13桁 or 20桁）
  barcodeType?: 'convenience'; // 現状はコンビニ収納用のみ
}
```

### プロンプト追記内容

```
もし書類にコンビニ払い用のバーコードが印刷されている場合、
バーコード下部に記載されている数値列（13桁または20桁の数字）を
`barcodeNumber` フィールドに抽出してください。
数値が読み取れない場合は null としてください。
```

## ディープリンク仕様

⚠️ **重要：実装前に必ず検証が必要**

PayPayおよびPayBのバーコード値を外部から渡すディープリンクは、**公式ドキュメントに記載が確認できていない**。以下は調査済みの事実と未検証事項の整理。

### 調査済みの事実

| 内容 | 確認状況 |
|------|----------|
| `paypay://` でPayPayアプリが起動する | ✅ 確認済み |
| `paypay://pay?barcode=` でバーコード値を渡せる | ❌ 公式未確認 |
| PayPayの請求書払いはアプリ内カメラスキャン設計 | ✅ 確認済み |
| `paybapp://` でPayBアプリが起動する | ⚠️ 未検証 |

### 実装方針（確認済み範囲のみ）

外部からバーコード値を渡す手段が確認できないため、現時点では**アプリ起動 + ユーザーへの手順案内**にとどめる。

```tsx
// components/DocumentDetail/PaymentActions.tsx

interface PaymentActionsProps {
  barcodeNumber: string | null;
}

export function PaymentActions({ barcodeNumber }: PaymentActionsProps) {
  if (!barcodeNumber) return null;

  return (
    <div className="mt-4 p-4 bg-yellow-50 border border-yellow-200 rounded-lg">
      <p className="text-sm text-yellow-800 mb-2">
        ⚠️ β機能 · 対応していない請求書の場合があります
      </p>
      <p className="text-sm text-gray-600 mb-3">
        PayPayの「請求書/税金」からバーコードをスキャンしてください
      </p>
      <div className="flex flex-col gap-2">
        {/* アプリ起動のみ。バーコード値の自動引き渡しは現時点で未対応 */}
        <a href="paypay://" className="w-full">
          <Button variant="default" className="w-full">
            PayPayを開く
          </Button>
        </a>
      </div>
    </div>
  );
}
```

### 今後の調査事項

実装前に以下を実機で検証すること：

1. `paypay://` でアプリが起動するか（iOS / Android 両方）
2. PayPay・PayB の非公開ディープリンクが存在するか（有志の検証記事・逆アセンブル情報等）
3. 各アプリの利用規約で外部からのディープリンク起動が禁止されていないか

## DBスキーマ変更

`documents` テーブルに以下カラムを追加:

```sql
ALTER TABLE documents
  ADD COLUMN barcode_number TEXT,         -- 収納用バーコード数値列
  ADD COLUMN barcode_type   TEXT,         -- 'convenience' 固定（将来拡張用）
  ADD COLUMN payment_method TEXT,         -- 'paypay' | 'payb' | 'convenience_store' | null
  ADD COLUMN paid_at        TIMESTAMPTZ;  -- 支払い済み日時（手動セット）
```

## UX設計

### 書類詳細ページの変更点

1. バーコードが抽出できた場合 → 「外部アプリで支払う (β)」セクションを表示
2. バーコードがない場合 → 何も表示しない（既存UI変更なし）
3. 支払い後 → ユーザーが「支払い済みにする」ボタンをタップ → ステータス更新

### ユーザーへの注意書き

- β機能であること
- 発行元によっては対応していない場合があること
- 支払い完了後は fumuly 側で「支払い済み」を手動セットする必要があること

## 免責事項（ユーザー向け表示文案）

### アプリ内表示テキスト

**β機能バナー（支払いボタン付近）**
```
この機能はβ版です。対応していない請求書の場合、
外部アプリ起動後に「非対応」と表示されることがあります。
```

**非対応だった場合の案内**
```
この請求書は現在、アプリからの支払いに対応していません。
お近くのコンビニエンスストアでお支払いください。
```

### 利用規約・ヘルプページ用の免責文（詳細版）

```
■ 外部アプリ連携による支払い機能について

fumuly の請求書払い連携機能（β）は、スキャンした払込票の
バーコード情報をPayPay・PayBなどの外部決済アプリに渡すことで、
コンビニに行かずに支払いを試みることができる補助機能です。

【支払いできない場合について】
払込票による支払いに対応するには、請求書の発行元（電力会社・
クレジットカード会社・自治体など）が各決済サービスの
収納代行ネットワークに加盟している必要があります。
これは fumuly の仕様ではなく、日本の収納代行の業界構造上の
制約であり、PayPay・PayB・モバイルレジなど
すべての請求書払いアプリで同様に発生します。

【fumuly が保証しないこと】
・特定の請求書が外部アプリで支払えること
・外部アプリでの支払い処理の成否・正確性
・外部アプリの障害・仕様変更による機能停止
・支払い後の領収書・確認書類の発行

【お支払いの最終確認】
支払いが正常に完了したかどうかは、必ず外部アプリ側の
取引履歴または請求元からの確認通知でご確認ください。
fumuly 上の「支払済み」ステータスはユーザーによる
手動設定であり、実際の支払い完了を保証するものではありません。

【対応できない場合】
本機能で支払いできなかった場合は、コンビニエンスストアの
レジにて払込票のバーコードを読み取ってお支払いください。
```
